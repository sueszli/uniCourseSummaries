Bitonic sequence (Definition)

Sequence (a0,a1,…,an−1)\small (a_0, a_1, \dots ,a_{n-1})(a0​,a1​,…,an−1​)﻿ is bitonic if

either there is an index i\small ii﻿ such that

a0≤a1≤⋯≤ai\small a_0 \leq a_1 \leq \dots \leq a_ia0​≤a1​≤⋯≤ai​﻿

ai+1≥ai+2≥⋯≥an−1\small a_{i+1} \geq a_{i+2} \geq \dots \geq a_{n-1}ai+1​≥ai+2​≥⋯≥an−1​﻿

or there is a cyclic shift such that the first case holds (first case holds if we just shift all elements to left or right)

Ordering bitonic sequences

= oblivious merge

bitonic_merge(int a[], int n) {
	if (n==1) {
		return;
	}

	// n is odd
	int nn = n/2;                 // offset
	int s = n/2;	                // size of (first half) a'

	if (n%2 == 1) {
		nn++;
		if (a[n/2] < a[n-1]) {      // a[n/2] will be in a' and a[n-1] in a''
			s++;
		}
	}

	for (i=0; i<n/2; i++) {       // parallelizable
		int mina, maxa;
		mina = min(a[i], a[i+nn]);
		maxa = max(a[i], a[i+nn]);
		a[i] = mina;                // (first half) a'
		a[i+nn] = maxa;             // (second half) a''
	}
	
	// recursive call of each half
	bitonic_merge(a,s);
	bitonic_merge(a+s,n-s);
}

Theorem

On a shared-memory system a bitonic sequence of length n\small nn﻿ can be sorted in

⌈log⁡2n⌉\small \lceil \log_2n\rceil⌈log2​n⌉﻿ parallel steps

O(nlog⁡n)\small O(n \log n)O(nlogn)﻿ operations

time of O((nlog⁡n)/p+log⁡p)\small O((n \log n) / p + \log p)O((nlogn)/p+logp)﻿

space of O(n)\small O(n)O(n)﻿ (this means it can be done in place)

The sorting is not stable.
