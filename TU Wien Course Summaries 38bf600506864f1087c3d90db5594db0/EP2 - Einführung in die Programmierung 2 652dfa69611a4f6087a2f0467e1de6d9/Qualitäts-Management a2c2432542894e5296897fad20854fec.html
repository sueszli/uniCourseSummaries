Design-by-Contract

Eignet sich nicht für Beweisführungen, Kommentare die Bedingungen für Methoden festlegen.

Objekte → „Server“

Methoden → „Service“

Methoden die Output beanspruchen → „Client“

Verträge zwischen Client und Server sind Zusicherungen (die meist als Kommentare) geschrieben sind.

Die Verantwortlichen für Bedingungen

  • Precondition - Client
  • Invariant - Server
  • Postcondition - Service

Hoare-Kalkül (Hoare Tripel)

Eignet sich für Beweisführungen. Bedingungen für spezifische Programmsegmente.

Struktur: {V} S {N}

Vorbedingung                       { x = 1 ; y = 2 }

Programmsegment                 x = y

Nachbedingung                    { x = 2 ; y = 2 }

Lässt sich hintereinander ausführen und verketten und damit partielle Korrektheit des Programmes beweisen.

Methode A:           {V} S1 {X}

Methode B:           {V} S2 {N}

Verkettung:            {V} S1 S2 {N}

Lässt sich im Interpreter mit assert Anweisungen formulieren

Wenn assert() nicht zutrifft wird die AssertionError Exception geworfen

Testen und Verbessern

Black-Box-Tests: unabhängig von Implementierung, von Anwendung hergeleitet

White-Box-Tests: vom Implementierung abgeleitet

Grey-Box-Tests: Testfälle vor Implementierung als Spezifikation festgelegt (Zwischending zwischen Blackbox und Whitebox)

Weitere Tests:

  • Regressions-Test (automatisiert)
  • Code-Review
  • Sicherheitstest (Pentest)
  • Belastungstest (Stresstest, Crashtest)
  • Benutzeroberflächentest

Hilfsmittel zum Finden von Fehlerursachen:

  • Code-Review
  • Debugger
  • Log-Dateien, Traces
  • Einbeziehen von Entwicklern, Experten, Nutzern

Mehrstufiges Testen:

  1. Modul- bzw. Komponententest
  1. Schnittstellentest
  1. Integrationstest
  1. Systemtest (auch Abnahmetest)

Qualität und Optimierung

Definition von Qualität:

  • Brauchbar (Aufgabenerfüllung, Bedienbarkeit, Konfigurierbarkeit, Attraktivität)
  • zuverlässig und sicher
  • effizient
  • wartbar

Qualität verbessern:

  • Qualitätsbewusstsein
  • Anforderungsanalyse
  • ständige Kontrolle
  • Besprechungen
  • Analyse-Werkzeuge
  • formale Beweise
