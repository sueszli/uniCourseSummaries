  • Programm-Ausführung

Batch processing

Menge an Jobs (= Batch) hintereinander ausgeführt:

  1. Job aus User Program Area von Eingabegerät lesen, Monitor verlassen
  1. JCL Instructions von Job interpretieren
  1. Programm ausführen
  1. Zurück zum Monitor, Wiederholen

Problem: Programme sind entweder CPU- oder IO-lastiger

Lösung: Multiprogramming

Multiprogramming

/ Multitasking / Multiprogrammed batch systems

Parallelisierung von CPU und IO mit Buffer und IO-interrupts:

Tasks (=Prozesse) benötigen Verwaltung von Speicher, CPU, Ressourcen.

Scheduling Während ein Programm auf IO wartet führen wir ein anderes aus

Non-Preemptive-Scheduling Keine Unterbrechung durch Scheduler

Time-Sharing Systems

Multi-User-System Job batches stückweise ausgeführt, kein User übernimmt CPU alleine

Preemptive-Scheduling Scheduler darf unterbrechen (zb bei IO-Input, Priorität, ...)

File System Leichterer Zugang zu Daten
