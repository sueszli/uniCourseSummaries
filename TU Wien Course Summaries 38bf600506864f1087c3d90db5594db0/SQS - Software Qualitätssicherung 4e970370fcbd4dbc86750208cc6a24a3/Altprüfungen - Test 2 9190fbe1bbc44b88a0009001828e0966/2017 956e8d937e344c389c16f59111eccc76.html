Äquivalenzklassen

Ein Online Lieferdienst für Lebensmittel führt ein Bonuspunktesystem ein.

Bestellungen sind als registrierter sowie als anonymer Kunde möglich.

Registrierte Kunden erhalten ab einer Einkaufssumme von ≥10€ nach dem Einkauf zwei Bonuspunkte gutgeschrieben.

Ab zehn Bonuspunkten werden diese automatisch eingelöst und die Einkaufssumme wird um 2€ reduziert.

Definieren Sie Anhand der gegebenen Intormationen alle Aquivalenzklassen und geben Sie zwei wichtige Testfälle (nicht JUnit-Tests) auf Basis von Grenzwerten an.

Registrierter Kunde
	Einkaufssumme >= 10 EUR
		>=8 Punkte -> Einkaufssumme -((Punkte+2)/10)*2 EUR, (Punkte+2)%10 Punkte
		<8 Punkte -> Einkaufssumme +0 EUR, +2 Punkte 
	Einkaufssumme < 10 EUR
		>=8 Punkte -> Einkaufssumme +0 EUR, +0 Punkte
		<8 Punkte -> Einkaufssumme +0 EUR, +0 Punkte

Anonymer Kunde
	Einkaufssumme >= 10 EUR
		>=8 Punkte -> Einkaufssumme +0 EUR, +0 Punkte
		<8 Punkte -> Einkaufssumme +0 EUR, +0 Punkte 
	Einkaufssumme < 10 EUR
		>=8 Punkte -> Einkaufssumme +0 EUR, +0 Punkte
		<8 Punkte -> Einkaufssumme +0 EUR, +0 Punkte

Zwei wichtige Testfälle (Grenzfälle)

Nein:

  1. Mocks sollen nur bei Komponententests eingesetzt werden, nicht bei Integrationtests.

    Bei Integrationtests würden Mocks keinen Sinn machen.

Mocking

Bei der Klasse AlarmService handelt es sich um ein Service, dass in einem gegebenen Zeitintervall eine Alarmanlage aktiviert.

interface Clock {
	// throws IllegalArgumentException if timeZoneOffset is not in range [+-24]
	Integer getDayTimeInMillis(Integer timeZoneOffset) throws IllegalArgumentException;
}

class AlarmService {
	private Clock clock;
	
	public void setClock(Clock clock) { this.clock = clock; }	

	public boolean isAlarmActive(int startHour, int endHour, boolean daylightSaving) {
		Integer dayTime = clock.getDayTimeInMillis(daylightSaving?1:0)/1000/60/60;

		if(startHour <= endHour)
			return (dayTime >= startHour) && (dayTime <= endHour);
		else
			return (dayTime <= startHour) && (dayTime >= endHour);
	}
}

Testen Sie die Methode isAlarmActive indem Sie die folgenden Testfälle in JUnit Syntax mit mocking implementieren (ohne Mocking Framework).

  1. startHour = 20, endHour = 6, daylightSaving = false, result = true
  1. startHour = 8, endHour = 16, daylightSaving = true, result = false

public class AlarmTest {
	public MockClock mc;
	public AlarmService as;

	@BeforeAll
	public void setup() {
		AlarmService as = new AlarmService();
		Clock mc = new MockClock();
		as.setClock(mc);
	}

	@Test
	public void testIsAlarmActive_shouldReturnTrueWithMock() {
		mc.setTime(16); // getDayTimeInMillis() always returns 16
		assertTrue(as.isAlarmActive(20,6,false));
	}

	@Test
	public void testIsAlarmActive_shouldReturnFalseWithMock() {
		mc.setTime(16); // getDayTimeInMillis() always returns 16
		assertFalse(as.isAlarmActive(8,16,true));
	}
}
public class MockClock {
	private Integer time;

	public void setTime(Integer time) { this.time = time; }

	public Integer getDayTimeInMillis(Integer timeZoneOffset) throws IllegalArgumentException {
		if (time == null) throw new IllegalStateException;
		if (timeZoneOffset > 24) throw new IllegalArgumentException;
		return this.time;
	}
}
