IFC

Goal: leaking the high variables to the low variables with the given type system

  • the code used to leak the values of the "high" variables h{1,6} to the corresponding "low" variables l{1,6}

Step 1

Program is secure / confidential if high inputs do not interfere with low inputs.

I leaked the information through an explicit flow (= Direct assignment) of every single variable.

l1 = h1;
l2 = h2;
l3 = h3;
l4 = h4;
l5 = h5;
l6 = h6;

Flag: WUT{e4sy_

Step 2

Implicit flows leak information through the control structure of the program.

In the given type system, implicit flows were allowed since the program did not check the type of the instruction guard.

if(h1){	l1 = true; } else { l1 = false; }
if(h2){	l2 = true; } else { l2 = false; }
if(h3){	l3 = true; } else { l3 = false; }
if(h4){	l4 = true; } else { l4 = false; }
if(h5){	l5 = true; } else { l5 = false; }
if(h6){	l6 = true; } else { l6 = false; }

Flag: ifc_

Step 3

This time we can not leak through an implicit flow.

Note: This step will be run only once, and always with the same secret. You will need to use this fact.

So now our goal must be to figure out the underlying boolean values. For this, we figure out the values by testing whether the program terminates or not.

Example:

while(h1){
	skip;
}

Therefore we directly assign:

l1 = false;
l2 = false;
l3 = true;
l4 = true;
l5 = true;
l6 = false;

Flag: ch4ll3ng3_

Step 4

Once again the underlying boolean values stay the same.

This time: loops with high guards are not allowed, loops inside ifs with high guards not allowed.

There are no timing channels that would reveal any information.

  • Loops must have LOW guards
  • no direct leak, no indirect leak
